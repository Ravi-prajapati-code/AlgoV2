# CIO Adversarial Review — "Prove This Has No Durable Alpha"

**Date**: 2026-07-07
**Framing**: Written from the position of a CIO whose job is to REJECT this strategy. Every issue
below is an argument *against* allocation. The strategy's defenders bear the burden of proof for
each. Issues are ordered by kill-shot severity, not by topic. Each carries: Evidence / Probability /
Impact / How to prove or disprove / Experiment to run.

**Scope**: All documents 01-16.5, plus session research records (scripts/, memory).

---

## Verdict up front

**The null hypothesis — "this strategy has no durable alpha" — cannot be rejected on the current
evidence.** Not because of one flaw, but because at least four independent kill-shots each
individually suffice, and they compound:

1. Every backtest number rests on a universe assembled with hindsight (Issue 1).
2. The out-of-sample window has been consumed by ~30+ candidate evaluations and is no longer
   out-of-sample (Issue 2).
3. ~~The strategy loses to its own equal-weight universe with negative IR — direct evidence that
   stock *selection* subtracts value~~ **RETRACTED 2026-07-07 after testing**: the permutation test
   (Issue 4, results below) shows the ranking DOES carry genuine selection information — the
   equal-weight shortfall is attributable to portfolio construction (3 slots, cash drag, timing
   costs) consuming that edge, not to anti-skill. This kill-shot is withdrawn.
4. Even taking every contaminated number at face value, a passive Midcap 150 ETF dominates on CAGR,
   Sharpe, Sortino, and Calmar (Issue 5) — and the factor-regression alpha is statistically zero
   (t=0.71), so no demonstrable net alpha exists after costs of construction.

Recommendation an IC would give: **no institutional capital**. Personal/limited capital only (per
docs/13's verdict, which this review reaffirms and strengthens), with forward paper-trading as the
only remaining clean test.

## Issue 1 — Universe look-ahead / survivorship bias contaminates every number (KILL SHOT)

**Evidence**: `docs/14_Universe_Verification_Report.md` and the live warning printed by every
backtest run: *"falling back to TODAY's static watchlist applied retroactively to 2022-01-01 —
this backtest's symbol selection may reflect look-ahead/survivorship bias."* Point-in-time
watchlist tracking begins 2026-07-06 — i.e., **two days ago**. 33 symbols were removed from the
watchlist using retrospective P&L/win-rate statistics (documented in the watchlist file's own
comments). The dynamic-universe subsystem separately leaked 26 hindsight-selected symbols
(recurring incident, `docs/08`). The one sensitivity test run (restoring the 33 removed symbols)
produced a **confounded** result — TEST window improved +6.93pp when losers were restored, opposite
of the survivorship prediction — meaning the project cannot currently even bound the bias's sign,
let alone its magnitude (`docs/14`, its own honest conclusion).

**Probability the reported numbers are materially inflated**: High (60-85%). A 100-symbol universe
curated in 2025-26 by someone who watched 2022-2026 unfold is the textbook mechanism; the removals
were explicitly performance-based.

**Impact**: Fatal to all quantitative claims. The honest baseline (+12.85% CAGR / 0.83 Sharpe)
already loses to the Midcap 150 ETF; any downward correction from decontamination widens the loss.
There is no floor under the numbers until this is fixed.

**How to prove/disprove**: Reconstruct the universe as it *would have been chosen* at each
historical date, using historical index constituent lists (NSE publishes Midcap 150 constituent
changes) or an as-of liquidity/market-cap screen — not today's watchlist.

**Experiment**: Rebuild a point-in-time universe for 2022-01-01 (and semi-annual rebalances),
re-run the full robustness gate. Separately, run the same-size-swap design docs/14 already
identified as needed (swap N hindsight-removed names for N random contemporaneous names of similar
liquidity) to isolate survivorship from pool-size effects.

## Issue 2 — Multiple testing has exhausted the out-of-sample window (KILL SHOT)

**Evidence**: `docs/05_Research.md` records at minimum: staged-entry, extension-filter, ATR sizing
(3x), correlation-aware sizing, hard drawdown gate, slot/allocation resize, "extensive re-tuning"
of regime hysteresis parameters (plural configurations, all evaluated), the 8-value position-count
sweep plus its robustness-gate run (2026-07-07), and the universe-restoration sensitivity test.
Every one of these was adjudicated against the **same** 2022-2024 train / 2025-present test split.
Conservatively 30+ distinct configurations have now seen the "out-of-sample" window. The
extension-filter case proves the test set has real adjudication power being spent: it *passed*
full-window and OOS and only died on stress scenarios.

**Probability**: Certain that the testing occurred (it is documented). Probability that the
surviving live configuration's test-window Sharpe is inflated by selection: High. With ~30 trials,
a deflated-Sharpe/Bonferroni-style haircut on a raw Sharpe of 0.83 over 4.5 years plausibly takes
the t-statistic below any rejection threshold.

**Impact**: The 2025-present window can no longer be cited as out-of-sample evidence for anything.
The project has **zero remaining clean holdout data**. Expected live Sharpe is materially below
backtest Sharpe.

**How to prove/disprove**: Count the true number of configurations evaluated (git history +
docs/05 + memory records make this auditable). Compute a deflated Sharpe ratio (Bailey &
López de Prado) or apply a White's-reality-check-style bootstrap across the tried configurations.

**Experiment**: (a) Deflated Sharpe with the honest trial count; (b) declare the existing data
fully spent and treat **forward paper-trading/live shadow for 6-12 months** as the only admissible
future evidence for the current configuration.

## Issue 3 — Statistical power: the evidence base cannot distinguish alpha from zero

**Evidence**: 156 trades over 4.5 years; 51 trades in the test window. The robustness gate's own
consistency check labels the baseline **UNSTABLE** (win rate 32.7% train vs 52.9% test — a 20pp
swing; the earlier OOS validation memo measured 19.3pp). A Sharpe of 0.83 measured over ~4.5 years
has a standard error of roughly 0.47 before any multiple-testing correction — the point estimate is
under 2 standard errors from zero, and after Issue 2's haircut, well under.

**Probability that true Sharpe is statistically indistinguishable from 0**: High.

**Impact**: Every downstream claim (asymmetric capture, Jensen's alpha, drawdown protection)
inherits this uncertainty. The IC cannot distinguish this track record from a lucky draw of a
zero-alpha, low-beta process.

**How to prove/disprove**: Confidence intervals, not point estimates.

**Experiment**: Moving-block bootstrap of daily strategy returns (10k resamples) → CI on Sharpe and
CAGR; trade-level bootstrap → CI on PF and win rate. Report whether zero is inside the interval.

## Issue 4 — Market timing disguised as stock selection (KILL SHOT for the hypothesis)

**Evidence**: The strategy loses to an equal-weight portfolio of **its own universe** by -4.71pp
CAGR with IR -0.25 and *worse* max drawdown (-23.67% vs -21.44%) despite beta ~0.5
(`docs/16`). Buying everything it screens, equally, with no selection skill, beat it. Meanwhile
beta 0.45-0.55 and down-capture 42-54% show the return profile is dominated by *when it is
invested* (the regime gate) rather than *what it picks*. Additionally, the safe-haven leg parked
capital in GOLDBEES during BEAR regimes across 2022-2026 — a historic gold bull market — so part of
the "protection alpha" is a one-off macro tailwind unlikely to repeat.

**Probability**: High. The equal-weight comparison is direct measurement, not inference.

**Impact**: The Gate 0 economic hypothesis (`docs/15` — cross-sectional momentum selection edge
from slow information diffusion) is contradicted by the project's own Gate 1 data. If the return
source is regime timing + gold, the strategy has made essentially **two macro calls** (avoid 2022
bear, hold gold), and two observations of timing skill is statistically nothing. There is then no
mechanism with a persistence argument behind it.

**How to prove/disprove**: Decompose returns into (a) the regime/exposure overlay applied to
passive holdings and (b) selection within the invested periods.

**Experiment**: Monte Carlo of random portfolios — same universe, same regime gate, same position
count and sizing, random picks from the screened candidates — 1,000 paths. If the actual strategy
lands inside the middle of that distribution, selection contributes nothing and the product is a
timing overlay. Separately, re-run the backtest with the safe-haven leg in cash instead of
GOLDBEES to isolate the gold windfall.

**RESULT (2026-07-07) — the strong form of this issue is REFUTED**:
`scripts/selection_skill_monte_carlo.py` ran the permutation test — 40 paths, each applying one
fixed symbol→symbol permutation to the full rs_rank history (preserving cross-sectional rank
distribution, temporal persistence, and daily threshold pass-counts; destroying only the link
between a stock's rank and its own prices; regime gate, trend filters, sizing, and safe-haven leg
identical in all arms). The actual strategy (CAGR +12.49%, Sharpe 0.41) beat **all 40** permuted
paths on both CAGR and Sharpe — permuted mean CAGR +2.90% (range -5.43% to +11.82%) — one-sided
permutation p = 0.024 on both metrics. The rs_rank ranking carries genuine selection information
worth roughly +9.6pp/yr over random selection under identical timing and construction. The correct
reading of the equal-weight-universe shortfall (docs/16) is therefore: **selection adds real value,
and the portfolio construction wrapper (3 concentrated slots, cash drag, timing whipsaw) consumes
more than selection adds.** The GOLDBEES→cash ablation (gold-windfall quantification) has still
not been run. Residual caveats: the rs_rank threshold and parameters were tuned on this same
window (Issue 2 applies to the magnitude, less so to the sign), and universe contamination affects
both arms but could still inflate a momentum ranking specifically if the surviving universe was
implicitly selected for momentum-friendliness (Issue 1).

## Issue 5 — Even at face value, a passive ETF dominates

**Evidence**: `docs/16`: vs Midcap 150 ETF — excess -5.07pp, IR -0.26, Sharpe 0.59 vs 0.85, Sortino
0.82 vs 1.07, Calmar 0.88 vs 1.01. The only benchmark it beats (Nifty 50) is the wrong asset class
for a midcap strategy. The position-count lever — the one concrete rescue hypothesis — was tested
2026-07-07 and closed (capital-dilution artifact; the N=4 variant failed the robustness gate
out-of-sample). The honest baseline also **fails the project's own validation gates** and is
running live anyway (`docs/05`, net conclusion).

**Probability**: Fact — already measured, using the *contaminated (favorable) data*.

**Impact**: This is the cleanest possible IC rejection: the best-case, look-ahead-assisted numbers
still lose to a zero-effort passive alternative on both raw and risk-adjusted terms.

**How to prove/disprove**: Nothing to disprove; the burden is fully reversed. The only remaining
defense is portfolio context — that the strategy adds value as a *component* (low correlation to
equity beta) rather than standalone.

**Experiment**: Mean-variance spanning test / simple two-asset frontier: does adding the strategy
to a Midcap-ETF portfolio improve the achievable Sharpe at any mix weight, using honest
(bootstrap-CI) return estimates? If not, there is no allocation, even as a diversifier.

## Issue 6 — Regime dependence: one cycle of evidence, and the stress tests already fail

**Evidence**: The entire dataset is 2022-2026: one bear (2022), one strong midcap bull (2023-24),
one choppy stretch (2025-26). All four synthetic stress scenarios produce catastrophic results for
baseline and every candidate alike: CAGR -27% to -39%, PF 0.28-0.46 (`robustness_gate` output,
2026-07-07 run). The regime gate does **not** protect in any synthetic bear — its good 2022 looks
like one favorable realization, not a demonstrated mechanism. Momentum crash risk is acknowledged
in `docs/15` itself as the known failure mode of this strategy class.

**Probability that performance is regime-specific**: High.

**Impact**: The "asymmetric protection" claim (down-capture < up-capture) rests on a single bear
market. If the next regime transition resembles any of the four stress scenarios, the realized
MDD materially exceeds the -23.67% in-sample figure.

**How to prove/disprove**: More cycles — backward or forward.

**Experiment**: Extend the backtest to 2016-2021 with a point-in-time universe (requires Issue 1's
fix first), explicitly including March 2020. If the regime gate protects there too, the timing
mechanism gains real support; if not, it was one lucky call.

## Issue 7 — Hidden factor exposure / crowding: the "alpha" may be buyable as an index

**Evidence**: No factor regression exists anywhere in the project — only single-benchmark CAPM.
The strategy is mechanically long the momentum factor in Indian midcaps during the largest retail
flow surge into midcap/momentum products in NSE history (2023-24 record SIP flows). Jensen's alpha
of +2.73%/yr vs the Midcap ETF is well within the range explained by a passive momentum-factor
tilt (an investable Nifty Midcap150 Momentum 50 index fund exists).

**Probability**: Medium-High.

**Impact**: If a two-factor regression (midcap beta + momentum factor) absorbs the alpha, the
strategy is a fee-and-effort-expensive replication of an index product — and it inherits momentum-
crowding crash risk it has never been tested against.

**How to prove/disprove**: Regress strategy daily returns on Midcap 150 returns plus a momentum
index return series; examine residual alpha and its t-stat.

**Experiment**: Two-factor (and optionally three: +smallcap) regression over the full window.
Compare strategy directly against the Midcap150 Momentum 50 index as an alternative benchmark row
in `docs/16`.

**RESULT (2026-07-07) — mixed; alpha statistically zero either way**:
`scripts/factor_regression.py`, common window 2022-08-11 → 2026-07-06 (n=965, constrained by
MOM30IETF.NS inception — note this window *excludes* the early-2022 bear where the regime gate
added value). One-factor (Midcap 150 ETF): alpha +5.40%/yr, **t=0.74**; beta 0.540. Two-factor
(+ Nifty200 Momentum 30 ETF): alpha +5.17%/yr, **t=0.71**; beta_midcap 0.547; beta_mom30 -0.008
(t=-0.82), R²=0.276. The momentum-factor loading is zero, so the "buyable as a momentum index"
version of this issue is not supported — with one material caveat: the midcap~mom30 daily return
correlation measured only 0.295, implausibly low for two long-only Indian equity ETFs, suggesting
MOM30IETF's thin trading attenuates the estimate (plain OLS, no HAC correction — stated
limitation). The more important number is the alpha t-stat: **under 1 in both specifications**,
before any multiple-testing haircut. Whatever selection information Issue 4's permutation test
found, it does not survive into statistically demonstrable net alpha versus passive exposure.

## Issue 9 — Benchmark construction flaws (cuts both directions, but unquantified)

**Evidence**: (a) Benchmarks are compared on **different date windows** in `docs/16` (Nifty 50 from
2022-01, Midcap ETF from 2022-03, Nifty 500 proxy only from 2023-10) — the rows are not directly
comparable to each other. (b) The Nifty 50 comparison — the only "win" — uses a price index against
what appears to be a dividend-inclusive strategy equity curve; adding ~1.2-1.5%/yr of dividends to
the benchmark cuts the +4.83pp excess meaningfully. (c) ETF proxies embed tracking error and were
identified best-effort, not broker-confirmed (`docs/14` decision record). (d) The risk-free rate is
a flat 6.5% assumption with no data source. (e) The equal-weight universe benchmark is built from
the same contaminated universe — though since the strategy *loses* to it anyway, decontamination
hurts the strategy's absolute numbers more than this comparison.

**Probability**: Fact that these constructions are as described; Medium probability any single one
changes a conclusion, but they all lean the same direction — flattering the strategy's one win.

**How to prove/disprove**: Recompute on a single common window using total-return index (TRI) data
for all benchmarks, and verify whether the backtest equity curve credits dividends.

**Experiment**: One table, one common window (2022-03 → present), all benchmarks as TRI, dividend
treatment stated explicitly on both sides.

## Issue 10 — Concentration: the track record may be a handful of trades

**Evidence**: Maximum 3 concurrent positions; 156 trades total; trade attribution (`docs/05`) shows
the edge concentrates in 31+ day holds — i.e., a small subset of long-duration winners. No
top-trade concentration analysis exists in any document.

**Probability that a small number of trades drive most of the P&L**: High (this is the typical
shape of trend-following P&L, and nothing here has been shown to differ).

**Impact**: If the top 5 trades account for the bulk of total return, the alpha estimate is
hostage to a handful of fills — a single missed entry (see Issue 8's live-fill gaps) rewrites the
track record. Luck and skill are indistinguishable at that concentration.

**How to prove/disprove**: Trivially measurable from the existing trade ledger.

**Experiment**: P&L concentration curve (cumulative % of net P&L vs trades ranked by P&L);
leave-out-top-5 and leave-out-top-10 re-computation of CAGR/Sharpe. Report what fraction of total
return survives.

## Issue 11 — The economic hypothesis is contradicted by the project's own data

**Evidence**: `docs/15` (Gate 0) claims a cross-sectional selection edge from slow information
diffusion in under-covered midcaps. `docs/16` (Gate 1) then measured selection: the strategy
underperforms an unranked, unselected, equal-weight basket of the identical universe, with negative
information ratio. The one documented empirical regularity (edge concentrated in 31+ day holds) is
equally consistent with generic trend exposure — it does not require, or evidence, selection skill.
`docs/15`'s own falsification list did not anticipate this failure mode (it only contemplated
alpha collapsing via high beta), so Gate 0 has been quietly failed without any document saying so.

**Probability that the stated mechanism is not the actual return source**: Medium-High.

**Impact**: Without a mechanism, there is no persistence argument — and "will it persist?" (Gate 2)
becomes unanswerable in principle, because there is nothing identified to persist. Gate 2 as
currently planned would measure the stability of numbers whose cause is unknown.

**How to prove/disprove**: Same decomposition as Issue 4 (random-selection Monte Carlo under
identical gating). If selection adds nothing, `docs/15` should be formally marked FALSIFIED and the
product re-scoped as a timing overlay — a different product with a different (and much harder)
burden of proof, per Issue 4's "two macro calls" problem.

**RESULT (2026-07-07) — Gate 0 mechanism partially VINDICATED, this issue substantially resolved**:
the permutation test (Issue 4 result) shows the cross-sectional RS ranking carries genuine
information — the actual ranking beat all 40 random permutations (p=0.024), adding ~9.6pp/yr CAGR
over random selection under identical gating. `docs/15`'s claimed mechanism (cross-sectional
momentum selection) is NOT falsified; the apparent contradiction with the equal-weight result is
resolved as a portfolio-construction problem, not a signal problem. Consequence: **Gate 2 (edge
persistence) is now a meaningful question again** — there is an identified signal whose stability
across sub-periods can be measured. The open research question shifts to the one the mandate doc
(16.5) anticipated: can construction be changed so the demonstrated selection information survives
into net returns? (Naively raising position count was already tested and rejected — capital
dilution, docs/16.)

## What would change this committee's mind

In priority order — cheapest and most decisive first:

1. **Trade-concentration + bootstrap CIs** (Issues 3, 10) — hours of work, existing data, could
   confirm immediately that the evidence base is too thin (or surprise us).
2. **Random-selection Monte Carlo under identical gating + GOLDBEES-to-cash ablation** (Issues 4,
   11) — decides whether there is any selection skill to defend at all.
   **→ DONE 2026-07-07 (MC half): broke IN THE STRATEGY'S FAVOR — selection information is real
   (beat all 40 permutations, p=0.024). GOLDBEES ablation still pending.**
3. **Factor regression vs Midcap Momentum index** (Issue 7) — decides whether the residual is
   buyable passively.
   **→ DONE 2026-07-07: no momentum-factor loading (thin-ETF caveat), but alpha t=0.71 —
   statistically zero net alpha stands.**
4. **Point-in-time universe reconstruction + full re-run** (Issue 1) — expensive, but nothing
   published from this project is defensible until it exists.
5. **Forward shadow period (6-12 months)** (Issue 2) — the only new clean evidence possible;
   everything historical is spent.

If steps 1-4 all break in the strategy's favor — selection skill exists, survives factor
adjustment, survives decontamination, and the P&L isn't five lucky trades — then and only then is
Gate 2 (edge persistence) worth running. On present evidence, the committee's expectation is that
at least one of them will not.
