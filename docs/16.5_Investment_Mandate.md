# Investment Mandate

**Investment Committee interlude, inserted before Gate 2 per explicit request.** `16_Benchmark_Attribution.md`
didn't just produce a pass/fail result — it exposed that this project has never stated what success
means. "CAGR = 12.85%" was treated as self-evidently good; Gate 1 shows that framing was doing more
work than it should have. Every claim below is labeled `[Fact / Opinion / Unknown, Confidence]`.

## The three candidate mandates, tested against real numbers

The working hypothesis going into this document was that the strategy might already satisfy one of
three mandates, and simply needed the framing to be named explicitly:

- **Mandate A — Beat Midcap 150 outright** (absolute return)
- **Mandate B — Better risk-adjusted return than the relevant passive alternative**
- **Mandate C — Smoother, lower-drawdown, institutional-quality equity curve** (capital preservation)

**[Fact, High confidence]**: measured against the two benchmarks that represent the strategy's
actual trading universe (Midcap 150 ETF, and its own equal-weight universe), **none of the three
currently hold**. The strategy loses on CAGR, Sharpe, Sortino, and Calmar against both. Against the
equal-weight universe, it even loses on Max Drawdown, despite roughly half the benchmark's beta. The
only benchmark against which all three mandates are simultaneously satisfied is Nifty 50 — a
large-cap index that doesn't match what this strategy actually trades.

**[Opinion, Medium confidence]**: this is a stronger and more useful finding than "we don't know
which mandate we're pursuing." It means the mandate question isn't currently "which of these have we
already won, we just need to name it" — it's "we haven't yet demonstrated a clear win under any
framing against the universe we actually operate in." That changes what this document needs to do:
not label an existing success, but set the bar future research is measured against.

## What is the objective?

**[Opinion, Medium confidence, proposed — requires the user's sign-off, not a technical decision]**:
**risk-adjusted return that survives institutional validation**, not absolute return. Reasoning: the
strategy's defining empirical property so far is low beta with small, inconsistent risk-adjusted
performance — not high raw return. Optimizing for CAGR alone (the project's original implicit goal)
would push toward higher position concentration and higher beta, which the evidence above suggests
would *increase* the very risk (idiosyncratic drag on Sharpe) already dragging down risk-adjusted
performance. An absolute-return mandate and this strategy's actual empirical behavior are in
tension; a risk-adjusted mandate is not.

## Who is the investor?

**[Unknown]** — not established anywhere in the project's documentation to date. This materially
changes which metrics matter: a retail-wealth mandate tolerates higher volatility for higher expected
terminal value; an institutional-allocation mandate (implied by the phrase "smoother institutional
equity curve" in the request that prompted this document) prioritizes drawdown control and Sharpe
even at the cost of CAGR. The metric priority order below assumes an institutional-allocation-style
investor, consistent with the language used to request this document — **flagged for confirmation,
not assumed silently**.

## Which benchmark matters?

**[Opinion, High confidence]**: **Nifty Midcap 150**, not Nifty 50 and not Nifty 500. The strategy's
own `03_Strategy.md` mechanism (cross-sectional relative-strength momentum among liquid mid-caps) and
its actual watchlist composition both point to mid-cap as the correct comparison class. Nifty 50 was
useful in `16_Benchmark_Attribution.md` as a sanity check (it's the one benchmark the strategy beats
cleanly), but using it as the primary yardstick would be comparing a mid-cap strategy's performance
against the wrong asset class and overstating the case. The equal-weight-universe benchmark is a
useful secondary check specifically for stock-selection skill (it isolates "did picking beat holding
everything," independent of asset-class drift) but inherits the [[universe-lookahead]] caveat from
`14_Universe_Verification_Report.md` and should not be treated as a fully independent benchmark.

## What risks are acceptable?

**[Opinion, Medium confidence, proposed]**: given the institutional-allocation framing, a maximum
drawdown ceiling meaningfully below the Midcap 150 ETF's own drawdown in the same period (currently
-20.81%, i.e. a target materially better than roughly -18%) is the natural acceptance bar — the
strategy currently misses this against its own equal-weight universe (-23.67%) though not against
the Midcap 150 ETF itself (-18.21%). Beta staying below 0.6 is consistent with what's already been
observed (0.45-0.55) and should be treated as a property to preserve, not just a byproduct — it is
the one metric where Gate 1 produced an unambiguous, desirable result.

## Which metrics define success, in priority order?

**[Opinion, Medium confidence, proposed]** — adopting the order suggested in the request that
prompted this document, since it matches the institutional-allocation framing above:

1. **Sharpe Ratio** (vs Midcap 150 ETF) — the primary go/no-go metric. Currently 0.59 vs
   benchmark's 0.85: **not met**.
2. **Calmar Ratio** (vs Midcap 150 ETF) — return per unit of max drawdown. Currently 0.88 vs
   benchmark's 1.01: **not met**.
3. **Max Drawdown** (vs Midcap 150 ETF, and separately vs own equal-weight universe) — currently
   better than the ETF (-18.21% vs -20.81%) but worse than the equal-weight universe (-23.67% vs
   -21.44%): **mixed**.
4. **Sortino Ratio** (vs Midcap 150 ETF) — currently 0.82 vs benchmark's 1.07: **not met**.
5. **CAGR** (vs Midcap 150 ETF) — deliberately ranked below the four risk-adjusted metrics, per the
   framing that raw return alone is not the objective. Currently +15.98% vs benchmark's +21.05%:
   **not met**.
6. **Win Rate** — retained as a diagnostic for mechanism health (per `17_Edge_Persistence.md`), not
   a success criterion in its own right.

**[Fact, High confidence]**: under this priority order, as measured today, the strategy does not
clear the bar on any of the first five metrics against the Midcap 150 benchmark. This is stated
plainly rather than softened — the purpose of defining the mandate now is to make results like this
legible, not to find a framing under which the current strategy already looks successful.

## Under what conditions should the strategy be retired or reworked?

**[Opinion, Medium confidence, proposed]**:

- If `17_Edge_Persistence.md` shows the mechanism (win rate, avg winner/loser, holding period) has
  materially shifted across sub-periods in a way inconsistent with the `15_Economic_Hypothesis.md`
  story — not just returns changing, but the *behavior* changing — that is a stronger retirement
  signal than a bad Sharpe number alone.
- **Tested 2026-07-07, condition triggered**: the concentration-risk hypothesis (idiosyncratic risk
  from the live `max_open_positions = 3` diluting a real systematic alpha; raising it should improve
  Sharpe) was tested by re-running the full backtest at position ceilings 3-20
  (`scripts/position_count_sensitivity.py`, results in `16_Benchmark_Attribution.md`). It does
  **not** explain the Sharpe gap as framed — trade count barely moves as the ceiling rises, so no
  real diversification occurs, and every metric collapses monotonically past N=4 for an unrelated
  reason (a capital-sizing artifact in `backtest/engine.py`, not idiosyncratic risk being diluted).
  Per this retirement condition, this **removes the plausible defense** of the current risk-adjusted
  shortfall — the mandate question this document opened is not resolved by this explanation, and
  remains open pending Gate 2. One unplanned lead did surface (N=4 beat the live N=3 on every metric
  in the full-window aggregate, including a Calmar exceeding the Midcap 150 benchmark's) but it was
  **REJECTED by `scripts/robustness_gate.py` on 2026-07-07**: N=4 wins on the 2022-2024 train split
  and then collapses on the 2025-present test split (Sharpe 0.39 vs baseline 0.87, PF 1.21 vs 1.76),
  the same full-window-hides-a-recent-breakdown pattern as
  [[out_of_sample_validation_20260703]] and the universe-restoration sensitivity test. **Position
  count is closed as a lever — live stays at 3.** Nothing tested so far explains the Sharpe shortfall
  against Midcap 150; the mandate question is still open into Gate 2.
- Positive Jensen's alpha alone, without Sharpe/Calmar/Sortino improvement, should **not** be treated
  as sufficient justification to continue running the strategy standalone — per the analysis above,
  it is consistent with a real but currently undelivered edge, not a delivered one.

## The reframed research direction

**[Opinion, Medium confidence]**: the combination of consistent low beta, small positive Jensen's
alpha, and Sharpe/Sortino/Calmar underperformance against the relevant benchmark is most consistent
with a strategy that behaves like a **low-exposure trend overlay with real but currently diluted
systematic value**, not a standalone return-maximizing product. The next research question this
raises is not primarily "does the edge persist" (Gate 2, next) but "does portfolio construction —
more positions, or blending this signal with others in a multi-strategy sleeve — unlock the return
that the low-beta/positive-alpha combination suggests exists, without losing the drawdown
characteristics that are still only partially demonstrated." This does not replace Gate 2; it gives
Gate 2 a sharper question to test.

## What Gate 2 should measure against, going forward

Per this mandate, `17_Edge_Persistence.md` should report Sharpe/Sortino/Calmar per sub-period against
Midcap 150 (not just CAGR/win-rate/holding-days per the original request), so that "does the edge
persist" is evaluated against the metrics that actually define success here — a strategy whose CAGR
is stable but whose Sharpe deteriorates over time has not demonstrated persistence under this
mandate, even though it would look stable under the original CAGR-first framing.
